# Revolut Live Coding Questions

> [!IMPORTANT]
> Thanks to ongoing feedback from candidates interviewing at Revolut, the material in this repository remains accurate and relevant for interview preparation as of **December 2025**.

This repository contains Python-based solutions to three core coding problems commonly encountered in Revolut's interview process:

- Load Balancer
- URL Shortener
- Ledger System

Each solution is thoroughly tested to ensure functional coverage and clarity.

Additionally, the repo includes an implementation of a money transfer system in SQL, showcasing different approaches such as:

- Transactions
- Optimistic locking
- Pessimistic locking

This mirrors Revolut's expectation of implementing core Ledger logic using pure SQL.

## Other Key Topics to Prepare for Revolut Interviews

To meet Revolut’s technical standards, you should also be familiar with the following topics:

- [CAP Theorem](https://en.wikipedia.org/wiki/CAP_theorem).
- [Domain-driven design](https://en.wikipedia.org/wiki/Domain-driven_design) (Bounded Contexts, Entities, Value Objects, Aggregates, Repositories, Domain Events).
- [Stability Patterns](https://medium.com/@maxi.gkd/stability-patterns-circuit-breakers-80aa69a06cb0).
- [Distributed Tracing](https://www.datadoghq.com/knowledge-center/distributed-tracing/).
- [Deployment strategies](https://www.apwide.com/8-deployment-strategies-explained-and-compared/) (Blue-Green, Canary, Feature flags, Shadow deployments).
- [Database migrations](https://en.wikipedia.org/wiki/Schema_migration) (Expand & Contract, Dual Write, Shadow Reads & Writes, Blue-Green, Online schema migration).
- [Database rollbacks](https://en.wikipedia.org/wiki/Rollback_(data_management)) (Transaction rollback, Backups).
- [ACID](https://en.wikipedia.org/wiki/ACID) principles and specifically [Isolation](https://en.wikipedia.org/wiki/Isolation_(database_systems)) levels.
- [Locking](https://en.wikipedia.org/wiki/Lock_(computer_science)) (Optimistic, pessimistic, Externalized).
- [Sharding](https://en.wikipedia.org/wiki/Shard_(database_architecture)), [Partitioning](https://en.wikipedia.org/wiki/Partition_(database)) and [Replication](https://en.wikipedia.org/wiki/Replication_(computing)).
- Architectures like [CQRS](https://en.wikipedia.org/wiki/Command_Query_Responsibility_Segregation) and [Event Sourcing](https://en.wikipedia.org/wiki/Event-driven_architecture).
- [Observability and Monitoring](https://www.dynatrace.com/news/blog/observability-vs-monitoring/).
